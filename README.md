## 📂 저장소 구조 및 제출 방법
`weekXX` 폴더 내에 **본인 이름 폴더**를 만들어 관리합니다.

### 1. 폴더 구조
```text
week01/
  ├── 본인이름/
  │     ├── BOJ_문제번호.py
  │     └── BOJ_문제번호.java
  └── 스터디원A/
        └── BOJ_문제번호.cpp
```

### 2. 파일 이름 규칙

`BOJ_문제번호.확장자` (예: `BOJ_1260.py`)

## 📝 화이트보드 인터뷰 대비
매주 **고정 문제**에 대해서는 아래 방식에 따라 로직 요약을 작성합니다.

### 1. 제출 방법
1. 주최자가 매주 생성하는 **GitHub Issue**(예: `[Week 01] 고정 문제 풀이 토론`)를 확인합니다.
2. 해당 이슈의 댓글(Comment)로 아래의 템플릿을 복사하여 작성합니다.
3. 인증 기한: 매주 월요일 24:00 (자정)까지 (코드 Push와 동일)

### 2. 화이트보드 인터뷰 대비 템플릿
*발표 당첨 시, 본인이 작성한 이 댓글을 화면 공유하며 설명합니다.*

```markdown
[BOJ_문제번호]
1. 제한 사항 및 시간 복잡도 분석
- 제한 사항: (예: N이 100,000이므로 O(NlogN) 이하의 알고리즘 필요)
- 예상 복잡도: (예: 이분 탐색을 활용한 O(NlogN))

2. 문제 해결 로직 (의사 코드 - Pseudocode)
    1. (예: 먼저 배열을 오름차순으로 정렬합니다.)
    2. (예: 시작점과 끝점을 설정하고 중간값을 비교합니다.)
    3. (예: 조건에 따라 범위를 좁히며 최적의 값을 찾습니다.)

3. 핵심 코드 설계
- 자료구조: (예: 효율적인 탐색을 위해 HashMap 사용)
- 엣지 케이스(Edge Case): (예: 입력값이 1개일 때, 모든 값이 같을 때, 결과값이 0일 때 등)

4. 실제 구현 시 어려웠던 구간
- (예: "인덱스 범위가 1 차이로 틀리기 쉬운 구간이었습니다.")
```
